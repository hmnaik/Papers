LLAH (Locally Likely Arrangement Hashing)

Locally Likely Arrangement Hashing (LLAH) is a kind of geometric feature descriptor. It takes advantages of affine invariants to achieve high accuracy and efficiency for recognition or retrieval of documents and images under affine transformations or small perspective distortions. It was first introduced to solve issues in document retrieval, and recently employed in building real-time Augmented Reality (AR) systems.

Following sessions will be served as a summary of what I learned so far, which will be guided by five papers in the order of publishing year.

1. 

Tomohiro Nakai, Koichi Kise, and Masakazu Iwamura. Use of Affine Invariants in Locally Likely Arrangement Hashing for Camera-Based Document Image Retrieval.Document Analysis Systems (DAS), 2006

In 2006, for the first time, Nakai et al. proposed this LLAH against Geometric Hashing (GH) to solve document retrieval problem with reduced computational cost and memory requirement. The affine invariant that LLAH utilizes is the ratio of the areas of two adjoined triangles made by four coplanr points. The feature point extraction part consists of several steps involving image processing: first binarize the image, and then find the connected components, and finally the centroids of each connected components are extracted as feature points. The feature vector is calculated at each feature point as following:  for each feature point, find its n  nearest points, then m  (m < n) points are chosen from those n points, and among these m points, four are chosen at a time to calculate the affine invariances. And this process is repeated for all combinations of chosing m from n and chosing 4 from m. Therefore, each feature point will have nCm descriptors and each descriptor is mC4 dimension. And each descriptor will undergo discretization and hashing to obtain a hash index. Finally the document ID, point ID and the discretized feature will be stored in a hash table according to the hash index. Details could be found in the paper.


2.

Tomohiro Nakai, Koichi Kise, and Masakazu Iwamura. Camera Based Document Image Retrieval with More Time and Memory Efficient LLAH. Second International Workshop on Camera-Based Document Analysis and Recognition (CBDAR), 2007

One year later, the same three guys have proposed an enhanced version of LLAH with less memory requirement and faster retrieval speed than the original one.

To reduce the required amount of memory, they remove features that would lead to collisions in the hash table, and let the hash table only store document ID and point ID (since there are at most one entry per index, there is no need to store the discretized feature values anymore).

To boost the speed of retrieval, they setup a rule to always find the same initial point when doing the cyclic permutations. Originally, for each feature point, nCm descriptors have to be calculated to achieve rotation invariance. But now, there are much less if we could ensure it always starts the permutation from the same initial point.

The experimental results show that “the proposed improvements realize reduction of the required amount of memory by about 80% and that of processing time by about 60%”. Noticeably, the accuracy also increases, and they suggest that it is due to less erroneous votes from the removed features.


3.

Hideaki Uchiyama and Hideo Saito. Random Dot Markers. IEEE Virtual Reality, 2011

Uchiyama et al. proposed a new marker system using random dots. So the features ARE the dots on the markers. And they use LLAH to calculate the feature vectors for registration and retrieval. And they argue that “Compared with traditional markers with square pattern, our random dot markers have several significant advantages for flexible marker design, robustness against occlusion and user interaction … We experimentally demonstrate that the discriminative ability of forty random dots per marker is applicable for retrieving up to one thousand markers.”


4.

Hideaki Uchiyama and Eric Marchand. Deformable Random Dot Markers. IEEE International Symposium on Mixed and Augmented Reality (ISMAR), 2011

A deformable version of the previously proposed random dot markers. May not be useful for our project right now, but it justifies LLAH's ability to handle non-planar surface.


5.

Hideaki Uchiyama and Eric Marchand. Toward Augmenting Everything:Detecting and Tracking Geometrical Features on Planar Objects. IEEE International Symposium on Mixed and Augmented Reality (ISMAR), 2011

In this paper, Uchiyama et al. try to use LLAH descriptor for retrieval of not only text documents but also various types of textures such as graffiti, handwriting, poster, ARToolKit marker and random dot marker. So the feature extraction stage is no longer just binarization since there could be only few connected components left for textured image. Instead, they turned to FAST corner detectors for feature extractions and employed an image pyramid with adaptively selected levels to ensure scale-invariance. For descriptor calculation and retrieval process, they are basically the same as previous.
